# AI获客工具 - 产品需求文档 (PRD)

## 1. 产品概述

一款面向直播运营/电商从业者的AI获客工具，通过在目标短视频下发布智能评论来引流到自己的账号主页。

**核心流程：** 筛选视频 → AI生成评论 → 多账号自动发布

## 2. MVP功能模块

### 2.1 视频筛选模块

**功能：** 根据关键词和筛选条件，抓取目标短视频列表

| 字段 | 说明 |
|------|------|
| 关键词 | 支持多关键词，用 `/` 分隔 |
| 平台 | 抖音、快手、小红书（多选标签） |
| 排序方式 | 最近发布 / 最多点赞 / 最多评论 |
| 最低点赞数 | 下拉选择：≥1000 / ≥5000 / ≥10000 |
| 评论活跃度 | 高（100条以上）/ 中（50-100）/ 低（50以下） |
| 粉丝画像 | 直播运营从业者 / 电商卖家 / 自媒体人 等 |

**输出：** 视频卡片列表，每个卡片显示：
- 平台标签 + 发布时间
- 视频标题
- 点赞数、评论数、转发数
- 选中状态

**操作：** 全选、单选、开始抓取

**统计：** 抓取总数、已评论、待处理

### 2.2 评论生成模块

**功能：** 对选中的视频，AI生成引流评论

**评论风格选项（单选）：**
- 业内共鸣：站在同行角度引发共鸣
- 提问好奇：以提问方式引发互动
- 经验分享：分享相关经验吸引关注

**备注要求：** 自由文本，补充生成指令

**生成规则：**
- 每个视频生成1条评论
- 评论需根据视频标题+平台特点+评论风格生成
- 评论要自然、不像广告、有引导性

**操作：** 复制、编辑、重新生成

### 2.3 账号管理模块

**功能：** 管理多个社交账号，控制发布节奏

**账号信息：**
- 账号名称、平台、头像/首字母
- 登录状态（在线/已过期）
- 今日已发/剩余配额

**发布节奏控制：**
- 每账号每日上限：10/15/20条
- 评论间隔：1-3分钟 / 3-8分钟 / 5-15分钟
- 账号轮换方式：顺序轮换 / 随机轮换

**发布队列：**
- 显示：账号、目标视频、评论内容、计划时间、状态
- 状态：发布中 / 待发布 / 已完成 / 账号异常
- 操作：取消、编辑、查看、重登录
- 全局操作：暂停全部 / 开始执行

**统计：** 账号总数、今日已发、剩余配额、账号在线数

## 3. 技术方案（MVP简化版）

### 3.1 技术栈
- **前端：** React + TypeScript + Tailwind CSS
- **状态管理：** Zustand
- **构建工具：** Vite
- **模拟数据：** 前端Mock（MVP阶段不接真实API）

### 3.2 MVP简化策略
- 视频抓取：使用模拟数据，预置一批视频
- AI评论生成：前端调用AI API（预留接口，先用模板生成）
- 账号管理：本地存储，模拟登录状态
- 发布功能：模拟发布流程，展示状态变化

### 3.3 后续迭代方向
- 接入真实平台API / 爬虫
- 接入大模型API生成评论
- 真实账号登录（Cookie/扫码）
- 真实评论发布
- 数据统计与分析面板

## 4. 页面结构

```
App
├── TopNav（视频筛选 | 评论生成 | 账号管理）
├── VideoFilterPage
│   ├── FilterSidebar（筛选条件）
│   └── VideoList（视频卡片列表 + 统计）
├── CommentGenPage
│   ├── VideoSelector（已选视频列表）
│   ├── StyleConfig（评论风格 + 备注）
│   └── CommentList（生成的评论列表）
└── AccountPage
    ├── AccountList（账号列表）
    ├── PublishControls（发布节奏控制）
    └── PublishQueue（发布队列表格）
```

## 5. 优先级

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P0 | 三个Tab页面框架 | 基础导航和布局 |
| P0 | 视频筛选展示 | Mock数据 + 筛选交互 |
| P0 | 评论生成展示 | 模板生成 + 编辑交互 |
| P0 | 账号管理展示 | 账号列表 + 发布队列 |
| P1 | 接入AI API | 真实AI生成评论 |
| P1 | 发布模拟 | 完整的发布状态流转 |
| P2 | 真实平台接入 | 爬虫/API |
